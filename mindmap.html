GameController
core.py
核心编排器。根据 AdvancedConfig 按需初始化 6 个高级组件，注入 GameLoop。工厂方法创建 AI 引擎。
GameLoop
game_loop.py
10 步增强循环。接收 6 个可选高级组件，每步按需调用。所有高级功能均可独立开关，关闭时走原有路径。

高级智能层 借鉴 Cradle 架构 - 5 大新模块

F1
TaskManager
task_manager.py
任务推断 + 子目标管理
维护 "当前在做什么" 的状态，避免每帧独立决策。跟踪任务切换历史，检测卡住并提醒 AI 换策略。
TaskStateSubGoal卡住检测
F2
ReflectionEngine
reflection.py
自我反思
操作前后截图对比，两层检查: 快速像素 diff（零成本）+ 区域变化分析。判断操作成败，生成调整建议。
3x3 区域分析自动重试调整建议
F3
Memory
memory.py
记忆系统
短期: 最近 N 帧滑动窗口，检测操作循环。
长期: ~/.gai_mcp/experiences/ 持久化 JSON，关键词检索，跨会话积累经验。
ShortTermMemoryLongTermMemory循环检测
F4
SkillManager
skill_manager.py
动态技能生成 & 检索
合并静态 .md 与 AI 动态生成的技能。按场景检索最相关技能，跟踪成功/失败率，自动淘汰差技能。
~/.gai_mcp/dynamic_skills/自动淘汰成功率追踪
F5
LocalAnalyzer
local_analyzer.py
分层决策（省 API 费）
本地像素分析处理简单帧（静止画面、对话推进），仅在场景变化大时调用 LLM。检测底部文字区域变化识别对话。
零 API 成本对话检测静态帧耐心
AdvancedConfig
models_advanced.py
全部 5 大功能的开关和参数。默认全部启用。
12 个配置项

升级对比

维度升级前升级后
决策方式每帧独立决策➞任务推断 + 子目标管理（TaskManager）
操作验证执行后不验证➞前后截图对比 + 区域分析（ReflectionEngine）
上下文记忆仅保留上一帧分析➞短期滑动窗口 + 长期持久化经验库（Memory）
技能系统静态 .md 文件➞静态 + AI 动态生成 + 自动淘汰（SkillManager）
API 调用每帧必调 LLM➞本地 CV 处理简单帧，按需调 LLM（LocalAnalyzer）
系统提示词基础操作指令➞+ 任务追踪 / 经验记录 / 技能生成指引
持久化存储仅配置文件➞+ 经验库 + 动态技能库（按游戏分文件）
